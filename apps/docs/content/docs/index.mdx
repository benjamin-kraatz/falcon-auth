# FALCON Auth

**FALCON Auth** is a unified, centralized authentication and identity provider platform.
Think of it like Google or GitHub SSO — except it's yours to own and control.

Any application can integrate FALCON Auth as an OAuth 2.1 / OIDC identity provider to authenticate users without managing credentials themselves.

## What FALCON Auth Provides

- **Identity Provider** — Acts as your own OAuth 2.1 / OIDC server
- **User Management** — Centralized user accounts, roles, and permissions
- **Organization Management** — Multi-tenant support with organizations and members
- **Admin Portal** — Full user and org management with impersonation capabilities
- **SDK** — Drop-in integration for any TypeScript/JavaScript application
- **Secure by default** — PKCE, rate limiting, session management built-in

## Quick Start

The fastest way to integrate FALCON Auth in your application:

```ts
import { createFalconAuthClient } from "@falcon-auth/sdk/client";

const auth = createFalconAuthClient({
  issuerUrl: "https://auth.falcon.example.com",
  clientId: "your-client-id",
  clientSecret: "your-client-secret",
  redirectUri: "https://yourapp.com/auth/callback",
});

// Generate an authorization URL and redirect the user
const { url, state, codeVerifier } = await auth.getAuthorizationUrl();
// Store state + codeVerifier, then redirect user to url
```

## Architecture

```
Your App  ──────────►  FALCON Auth Server (OAuth 2.1 / OIDC)
                              │
                    ┌─────────┴─────────┐
                    │                   │
               Auth API           Admin Portal
             (Better Auth)       (Web App)
                    │
               Turso DB (SQLite)
```

FALCON Auth is built on:
- **[better-auth](https://better-auth.com)** — The authentication engine
- **Hono** — Lightweight server framework
- **Drizzle ORM** — Type-safe database access
- **TanStack Start** — Admin portal framework
- **Cloudflare Workers** — Deployment platform
